## 컴퓨터 구조학습 1강

### 1. 컴퓨터 구조를 학습해야 하는 이유
- 개발자가 프로그래밍 언어의 문법을 넘어 컴퓨터 구조를 이해해야 하는 이유는 크게 두 가지 관점으로 요약
#### 1.1 문제 해결 능력의 배양
- 분석 대상으로서의 컴퓨터: 코드에 문제가 없음에도 실행 환경에 따라 오류가 발생하는 경우, 컴퓨터 내부를 들여다볼 수 있는 능력이 필요. 컴퓨터 구조를 이해하면 컴퓨터를 미지의 대상이 아닌 분석의 대상 으로 인식하게 되어 근원적인 문제를 해결가능.
- 채용 시장의 요구: 구글, 삼성전자와 같은 주요 IT 기업에서는 컴퓨터 구조에 대한 이해를 필수 역량으로 요구. 특히 하드웨어 밀접도가 높은 직무에서는 해당 과목의 이수 여부가 직무 수행 능력의 척도.
#### 1.2 성능, 용량, 비용을 고려한 개발

- 서버 컴퓨터나 클라우드 서비스를 이용할 일이 생길때 무조건 최신 또는 저렴한 장비를 고르는 것이 아니라, 개발한 프로그램의 특성에 맞는 CPU, 메모리, 스토리지의 성능과 용량을 판단할 수 있어야 한정된 비용에서 최대 효율을 내는 선택의 안목을 기를수있음.
- 효율적인 프로그래밍: 컴퓨터 구조를 알면 자원의 한계를 인지하고, 성능과 비용의 균형을 맞춘 효율적인 코드를 작성할 수 있음.

### 1. 컴퓨터가 이해하는 정보: 데이터와 명령어

컴퓨터는 모든 정보를 0과 1로 이해하며, 그 정보는 크게 두 종류로 나뉨.

| 구분 | 정의 | 특징 |
|---|---|---|
| 데이터 (Data) | 숫자, 문자, 이미지, 동영상 등 정적인 정보 | 명령어를 실행하기 위한 일종의 '재료' 역할을 함 |
| 명령어 (Instruction) | 컴퓨터를 실질적으로 움직이는 정보 | 데이터(재료)를 가공하여 특정 동작을 수행하게 함 |




### 2. 컴퓨터의 4가지 핵심 부품

컴퓨터의 종류와 상관없이 내부에는 동일한 역할을 수행하는 네 가지 핵심 부품이 존재합니다.

#### 2.1 메모리 (주기억장치)

- 역할: 현재 실행되는 프로그램의 명령어와 데이터를 저장합니다.

- 특징:
  - 프로그램이 실행되려면 반드시 메모리에 저장되어 있어야 함.

  - 주소(Address): 메모리 내의 특정 위치를 식별하기 위해 사용됨.

  - 휘발성: 전원이 꺼지면 저장된 내용이 사라짐 (주로 RAM을 지칭).

#### 2.2 CPU (중앙처리장치)

메모리에 저장된 명령어를 읽어 들이고, 해석하고, 실행하는 '컴퓨터의 두뇌', 내부적으로 세 가지 핵심 구성 요소를 가짐.

- ALU (산술논리연산장치): 계산기 역할을 수행하며 산술 및 논리 연산을 처리.

- 레지스터 (Register): CPU 내부의 작은 임시 저장 장치로, 명령어와 데이터를 실행하는 동안 일시적으로 보관.

- 제어장치 (Control Unit): 제어 신호(전기 신호)를 내보내어 부품을 관리하고 명령어를 해석.

#### 2.3 보조기억장치

- 역할: 전원이 꺼져도 정보를 보관해야 하는 프로그램을 저장. (예: SSD, HDD, USB 등)

- 관계: 메모리는 '실행할 프로그램'을, 보조기억장치는 '보관할 프로그램'을 저장.

#### 2.4 입출력장치

- 역할: 컴퓨터 외부와 내부가 정보를 주고받을 수 있게 하는 장치. (예: 마이크, 키보드, 모니터 등)

- 참고: 보조기억장치와 입출력장치는 '주변 장치'라는 개념으로 통칭되기도 함.


### 3. 메인보드와 시스템 버스

핵심 부품들은 메인보드(마더보드)에 장착되어 서로 정보를 주고받으며, 이 통로를 *버스(Bus)*라고 함.

#### 3.1 시스템 버스의 구성

핵심 부품 간의 소통을 담당하는 가장 중요한 통로인 시스템 버스는 세 가지로 나뉨.

- 주소 버스 (Address Bus): 주소를 주고받는 통로.

- 데이터 버스 (Data Bus): 명령어와 데이터를 주고받는 통로.

- 제어 버스 (Control Bus): 제어 신호를 주고받는 통로.
