# 강의 9강-10강
-------

# 1. ALU (산술논리연산장치):

ALU는 CPU 내부에서 실질적인 연산을 담당하는 부품입니다. 하지만 이를 단순히 '수식을 푸는 장치'로만 이해해서는 부족합니다.  
ALU는 레지스터와 제어장치 사이에서 데이터 처리를 수행하는 핵심 허브입니다.

---

## 입출력 구조 분석

ALU가 연산을 수행하기 위해 받아들이고 내보내는 정보는 다음과 같습니다.

### 입력 정보
- **피연산자(Operand)**:  
  레지스터로부터 전달받은 연산 대상 데이터 (숫자, 문자, 주소 등)
- **제어 신호**:  
  제어장치로부터 전달받은 '수행할 연산(덧셈, 뺄셈 등)'에 대한 지시

### 출력 정보
- **연산 결과**:  
  계산된 값으로, 메모리보다 훨씬 빠른 레지스터에 우선 저장됨
- **플래그(Flag)**:  
  연산 결과에 대한 '상태 리포트'

---

## ALU 내부 구성 (개념적 이해)

비록 회로 수준까지 깊게 들어가지는 않지만, ALU 내부에는 다음과 같은 전용 계산 회로들이 존재합니다:

- 가산기 (덧셈)
- 보수기 (뺄셈)
- 시프터 (비트 이동)

이 회로들이 유기적으로 동작하여 연산을 수행합니다.

---

## 플래그(Flag)와 흐름 제어의 연결

플래그는 단순한 정보 기록을 넘어 CPU의 다음 동작을 결정하는 중요한 요소입니다.  
플래그 레지스터에 저장된 값은 제어장치로 전달되어 조건부 명령(JUMP, CALL 등)의 실행 여부를 판단하는 근거가 됩니다.

### 주요 플래그 종류

- **부호 플래그 (Sign Flag)**  
  → 결과가 양수(0)인지 음수(1)인지 표시

- **제로 플래그 (Zero Flag)**  
  → 결과가 0인지 여부 (조건문 판단의 핵심)

- **캐리 플래그 (Carry Flag)**  
  → 올림수 또는 빌림수 발생 여부

- **오버플로우 플래그 (Overflow Flag)**  
  → 연산 결과가 레지스터 용량 초과 여부

- **인터럽트 플래그 (Interrupt Flag)**  
  → 인터럽트 처리 가능 여부

- **슈퍼바이저 플래그 (Supervisor Flag)**  
  → 현재 실행 모드 (커널 모드 / 사용자 모드)

---

# 2. 제어장치 (Control Unit): 컴퓨터의 오케스트라 지휘자

제어장치는 컴퓨터의 모든 부품을 일사불란하게 움직이게 만드는 '지휘소'입니다.  
시스템 전체의 동기화와 원활한 명령어 실행을 보장하는 핵심 역할을 수행합니다.

---

## 입력 신호와 동기화의 원리

제어장치는 다음 4가지 요소를 기반으로 제어 신호를 생성합니다:

- **클럭 (Clock)**  
  → 컴퓨터의 '심장 박동'  
  → 모든 명령어 실행 단계(가져오기, 해석, 실행 등)를 동기화

- **해석할 명령어 (Instruction)**  
  → 명령어 레지스터(IR)에 저장된 명령 정보

- **플래그 값 (Flags)**  
  → ALU 연산 결과를 기반으로 조건 판단

- **외부 제어 신호 (External Signals)**  
  → I/O 장치 등 외부에서 전달되는 요청

---

## 제어 신호의 분류 (Output)

제어장치가 생성하는 신호는 대상에 따라 다음과 같이 구분됩니다:

| 분류 | 대상 부품 | 주요 목적 |
|------|----------|----------|
| CPU 내부 | ALU | 산술/논리 연산 종류 지시 |
| CPU 내부 | 레지스터 | 데이터 이동 및 저장 제어 |
| CPU 외부 | 메모리 | 읽기(Read) / 쓰기(Write) 제어 |
| CPU 외부 | I/O 장치 | 입출력 및 상태 테스트 제어 |
