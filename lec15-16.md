# 2. RAM의 물리적 특성과 성능의 상관관계
메모리의 중추인 RAM(Random Access Memory)은 휘발성(Volatility)이라는 물리적 한계를 가짐. 이는 전원이 차단되면 데이터가 소멸되는 특성으로, 비휘발성인 보조기억장치와 대조됩니다. 이 차이로 인해 시스템은 '보관할 대상'은 보조기억장치에, 실행할 대상은 RAM에 두는 이원적 구조를 취하게 됩니다.
RAM의 용량이 시스템 성능에 미치는 영향은 '책상(RAM)과 책장(보조기억장치)'의 비유로 명확히 분석할 수 있습니다.
성능의 본질: 책상이 넓으면(RAM 용량이 크면) 한꺼번에 많은 책(프로그램)을 펼쳐놓고 작업할 수 있습니다. 이는 필요한 정보를 찾기 위해 매번 멀리 떨어진 책장(보조기억장치)을 오가는 수고를 혁신적으로 줄여줍니다.
시스템적 이득 (So What?): RAM 용량 증설의 진정한 가치는 단순히 속도가 빨라지는 것이 아니라, 보조기억장치의 치명적인 약점인 지연 시간(Latency)을 은폐(Hiding)한다는 데 있습니다. CPU가 느린 보조기억장치의 응답을 기다리며 유휴 상태에 빠지는 병목 현상을 최소화함으로써, 현대 컴퓨팅의 핵심인 다중 작업(Multitasking)의 효율성을 극대화할 수 있습니다.

# RAM의 유형별 비교 분석: DRAM vs SRAM
- 메모리는 설계 방식과 용도에 따라 서로 다른 기술적 경로를 걸어왔습니다. 전하의 방전을 막기 위해 끊임없이 재충전이 필요한 DRAM과, 전원 공급 시 상태를 안정적으로 유지하는 SRAM이 그 중심에 있습니다.

DRAM은 구조가 단순하여 오밀조밀하게 설계(높은 집적도)할 수 있어 대용량 주메모리에 적합합니다. 반면 SRAM은 매우 빠르지만 집적도가 낮고 가격이 비싸, 대용량화에는 한계가 있습니다. 이러한 특성 때문에 SRAM은 용량보다는 속도가 절대적으로 중요한 CPU 내부의 '캐시 메모리'로 특화되어 사용됩니다.

# 4. 대역폭 확장을 통한 성능 혁신: SDRAM에서 DDR4까지
- 초기 DRAM은 CPU의 클럭(Clock) 속도를 따라가지 못해 전송 효율이 낮았습니다. 이를 극복하기 위해 등장한 **SDRAM(Synchronous DRAM)**은 클럭 신호와 박자를 맞춰 데이터를 전송하며 혁신을 가져왔습니다.
이후 기술은 대역폭(Bandwidth), 즉 데이터 도로의 너비를 확장하는 방향으로 진화했습니다.
SDR(Single Data Rate): 클럭 한 주기에 한 번 데이터를 주고받는 1차선 도로입니다.
DDR(Double Data Rate): 도로의 너비를 확장하여 한 주기당 두 번의 데이터를 전송함으로써 대역폭을 2배로 늘렸습니다.
세대별 진화 (2^n 법칙): DDR 기술은 세대를 거듭하며 대역폭을 이전 세대 대비 2배씩 기하급수적으로 확장해 왔습니다. **SDR(1배) → DDR(2배) → DDR2(4배) → DDR3(8배) → DDR4(16배)**의 논리적 구조를 갖습니다.
현재 주류인 DDR4는 초기 표준 대비 16배의 대역폭을 제공하며 고해상도 영상 및 대규모 데이터 처리가 요구되는 현대 시스템의 성능 병목을 해결하는 핵심 동력이 되고 있습니다.

# 5. 메모리 주소 체계의 이원화: 물리 주소와 논리 주소
시스템은 하드웨어와 소프트웨어의 독립적인 운영을 위해 주소 체계를 이원화합니다.
물리 주소(Physical Address): 메모리 하드웨어가 실제로 인식하는 실제 번지입니다.
논리 주소(Logical Address): CPU와 각 프로그램이 인식하는 독자적인 주소 공간입니다. 모든 프로그램은 0번지부터 시작하는 자신만의 논리 주소를 가집니다.
이러한 분리 덕분에 소프트웨어 개발자는 사용자의 컴퓨터 환경이나 메모리 적재 위치를 고민할 필요 없이 독립적으로 프로그램을 작성할 수 있습니다. 운영체제가 이를 물리 공간에 유연하게 적재함으로써 자원 관리의 효율성을 보장하기 때문입니다.
